## Notes for Report Writing

The following points are worth discussing in the relevant report sections:

### Implementation section
- Evaluation ran 25 questions across 3 models in 329.8s
- Rate limiting required exponential backoff with jitter — implemented in `base_client.py`
- Gemini SDK was deprecated mid-project; migrated from `google-generativeai` to `google-genai`
- `temperature=0` set on all models to ensure deterministic, reproducible responses

### Scoring/evaluation section
- Three scoring modes used: exact match (mathematics), keyword matching (factual/logical), rubric (ethics/creative)
- Rubric scoring for subjective domains uses keyword presence + response length as proxies for quality
- This is a known limitation — discuss in critical evaluation section

### Results/findings section
- GPT-4o-mini shows domain-specific behaviour: strong in Mathematics, weaker in Logical Reasoning
- Gemini 2.5 Flash shows domain-specific behaviour: strong in Mathematics, weaker in Ethical Reasoning
- Claude Haiku 3 shows domain-specific behaviour: strong in Factual Accuracy, weaker in Ethical Reasoning
- The Ethical Reasoning domain shows the most variation between models (69.8pp gap), suggesting this domain most effectively differentiates model capabilities

### Critical evaluation section
- No single model dominated across all domains — supports the project's core argument that domain-specific evaluation is more useful than a single overall score
- Scoring methodology for ethics/creative is heuristic-based and should be acknowledged as a limitation
- Results from a single run may not be fully reproducible; multiple runs would improve statistical validity
- Dataset of 200 questions, while sufficient for demonstration, is smaller than established benchmarks (MMLU: 14,000+ questions)
